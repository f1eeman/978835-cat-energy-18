# feat: анимировать иконки кошки и пумы в секции каталогов

**Labels:** `ready-for-agent`

---

## Problem Statement

В секции «Похудение» и «Набор массы» на главной странице есть SVG-иконки кошки и пумы, вставленные как CSS `background-image` на `::after`-псевдоэлементах. Они статичны. Посетитель видит неподвижный силуэт — упущена возможность оживить блоки и подчеркнуть смысл (похудение → прыжок/лёгкость; набор массы → мощный бег).

## Solution

Перенести оба SVG из CSS `background-image` в inline-элементы HTML, чтобы GSAP мог управлять ими напрямую.

- **Кошка (Slim / Похудение):** прыгает вверх → зависает → приземляется → сидит → пауза → снова прыгает. Бесконечный цикл.
- **Пума (Pro / Набор массы):** горизонтальный беговой цикл — движется вперёд с лёгким отскоком вверх, зациклено.

Обе анимации реализуются через GSAP `timeline` с `repeat: -1`, уже подключённый на странице (`window.gsap`).

## User Stories

1. Как посетитель главной страницы, я хочу видеть прыгающую кошку в блоке «Похудение», чтобы визуально ощутить лёгкость и энергичность программы.
2. Как посетитель, я хочу видеть бегущую пуму в блоке «Набор массы», чтобы почувствовать силу и динамику продукта Cat Energy Pro.
3. Как посетитель, я хочу, чтобы кошка после прыжка возвращалась в сидящую позу и снова прыгала по кругу, чтобы анимация не была монотонной.
4. Как посетитель, я хочу, чтобы пума непрерывно бежала по кругу (двигалась вперёд, слегка подпрыгивала), чтобы блок выглядел живым.
5. Как пользователь с настройкой `prefers-reduced-motion`, я хочу, чтобы анимации отключались, чтобы страница не вызывала дискомфорта.
6. Как посетитель на мобильном устройстве, я хочу, чтобы анимации не потребляли лишних ресурсов CPU, чтобы страница работала плавно.
7. Как посетитель без JavaScript, я хочу видеть статичные иконки кошки и пумы, чтобы содержимое блоков оставалось понятным.
8. Как разработчик, я хочу, чтобы SVG-иконки были встроены inline в HTML, а не дублировались в SCSS, чтобы источник истины был один.

## Implementation Decisions

- **Перенос SVG в HTML:** из CSS `background-image` в `::after` → в inline `<svg>` элементы внутри `.catalogues__item`. Каждая иконка получает класс `product-package__cat` + модификатор (`product-package__cat--slim` и `product-package__cat--pro`).

- **Изменение CSS:** убрать `background-image` из `.product-package--losing-weight::after` и `.product-package--muscle-gain::after`. Сохранить `::before` (зелёный круг) без изменений. Перенести позиционирование (top/left/right/width/height) из `::after` в новый класс `.product-package__cat`, адаптируя под все брейкпоинты (mobile/tablet/desktop) — те же координаты, что были у `::after`.

- **GSAP-анимация кошки (slim):** использовать `gsap.timeline({ repeat: -1 })`:
  1. прыжок вверх: `y: -40px, duration: 0.35, ease: 'power2.out'`
  2. зависание на пике: `y: -40px, duration: 0.15, ease: 'none'`
  3. приземление: `y: 0, duration: 0.3, ease: 'bounce.out'`
  4. пауза в сидящей позе: `y: 0, duration: 1.2, ease: 'none'`
  Затем цикл повторяется.

- **GSAP-анимация пумы (pro):** использовать `gsap.timeline({ repeat: -1 })`:
  1. шаг вперёд с отскоком: `x: +12px, y: -6px, duration: 0.22, ease: 'sine.inOut'`
  2. шаг обратно к центру: `x: 0, y: 0, duration: 0.22, ease: 'sine.inOut'`
  3. небольшой шаг назад: `x: -5px, y: -3px, duration: 0.18, ease: 'sine.inOut'`
  4. возврат: `x: 0, y: 0, duration: 0.18, ease: 'sine.inOut'`
  Вся последовательность зациклена, создавая ощущение бегущего шага.

- **`prefers-reduced-motion`:** та же проверка, что уже есть в `scripts.js` (`window.matchMedia('(prefers-reduced-motion: reduce)').matches`) — если true, анимации не запускаются.

- **Место кода:** вся логика добавляется в `scripts.js` — новая секция после существующих анимаций промо-секции.

- **`aria-hidden="true"`** на inline SVG — иконки декоративные, не несут смысловой нагрузки для скринридеров.

### Анимационные тайминги (из прототипа)

```
Cat slim timeline (repeat: -1):
  jump up   → y: -40, 0.35s, power2.out
  hang      → y: -40, 0.15s, none
  land      → y:   0, 0.30s, bounce.out
  sit pause → y:   0, 1.20s, none

Puma pro timeline (repeat: -1):
  step fwd  → x:+12, y:-6,  0.22s, sine.inOut
  center    → x:  0, y: 0,  0.22s, sine.inOut
  step back → x: -5, y:-3,  0.18s, sine.inOut
  center    → x:  0, y: 0,  0.18s, sine.inOut
```

## Testing Decisions

Тестирование ручное, без автоматизированного фреймворка.

Хороший тест проверяет видимое поведение, а не детали GSAP-кода.

Ручной чеклист для приёмки:

- [ ] Открыть `build/index.html`. Оба SVG видны на своих местах (кошка в блоке «Похудение», пума в блоке «Набор массы»).
- [ ] Кошка выполняет прыжок, зависает, приземляется и сидит перед следующим прыжком — цикл бесконечный.
- [ ] Пума совершает беговые шаги (покачивание вперёд-назад с лёгким отскоком) — цикл бесконечный.
- [ ] Иконки позиционированы так же, как было до изменений (на зелёном круге `::before`).
- [ ] Эмулировать `prefers-reduced-motion: reduce` в DevTools → анимации остановлены, иконки статичны.
- [ ] Проверить на мобильной ширине (375px) — иконки видны, не выходят за пределы блоков.
- [ ] Отключить JS → иконки видны статично, ошибок нет.
- [ ] `catalog.html` и `form.html` — этих иконок нет, ошибок в консоли нет.

## Out of Scope

- Покадровая скелетная анимация (анимация отдельных лап, тела) — SVG состоит из одного плоского пути `<path>`, разбить его на части невозможно без редизайна иллюстрации.
- Создание новых SVG-иллюстраций или замена текущих.
- Анимация на страницах `catalog.html` и `form.html`.
- Звуковые эффекты.

## Further Notes

Оба SVG (`icon-cat-slim.svg` и `icon-cat-pro.svg`) — одиночные `<path>`-элементы в едином цвете. Это означает, что доступна только анимация трансформаций всего SVG целиком (translate, scale, rotate). Тайминги в разделе Implementation Decisions были подобраны так, чтобы анимация читалась как прыжок / бег на основе движения цельного силуэта.

GSAP и `window.gsap` уже доступны на странице (загружены в `scripts.js` после добавленной анимации промо-секции в предыдущем коммите).
