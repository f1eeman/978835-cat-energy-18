# feat: убрать 3D-кота из промо-секции, добавить GSAP-анимацию банки с искорками

**Labels:** `ready-for-agent`

---

## Problem Statement

Промо-секция главной страницы использует Three.js для рендера 3D-модели кота в `<canvas>`. Это создаёт несколько проблем: Three.js — тяжёлая зависимость (~600 КБ vendor-файлов), которая загружается только ради одного декоративного элемента; 3D-кот визуально не соответствует стилю сайта (плоский, low-poly — контрастирует с графикой продукта); `cat3d.js` при загрузке скрывает `<picture>`-изображение банки, лишая пользователей основного визуального якоря страницы; на медленных соединениях и слабых устройствах WebGL может не запуститься или тормозить.

## Solution

Удалить 3D-кота (`cat3d.js`, Three.js vendor-файлы, `<canvas id="cat3d">`). Восстановить видимость существующего `<picture>` с банкой продукта. Добавить лёгкую GSAP-анимацию: банка плавно покачивается (floating) с лёгким наклоном. Вокруг банки разместить 5–6 inline-SVG искорок (`✦`-форма, 4-лучевая звезда), каждая анимируется GSAP — пульсация масштаба и прозрачности со стаггером. Итог: живая, брендовая анимация без WebGL, без тяжёлых зависимостей.

## User Stories

1. Как посетитель главной страницы, я хочу видеть изображение продукта (банку Cat Energy) в промо-секции, чтобы сразу понять, что продаётся.
2. Как посетитель, я хочу видеть ненавязчивую анимацию в промо-секции, чтобы страница выглядела живой и привлекательной.
3. Как пользователь слабого устройства или медленного соединения, я хочу, чтобы анимация не требовала WebGL, чтобы страница работала плавно на любом устройстве.
4. Как мобильный пользователь, я хочу, чтобы анимация не потребляла много ресурсов CPU/GPU, чтобы телефон не перегревался.
5. Как посетитель, я хочу, чтобы анимация не отвлекала от текста («Функциональное питание для котов») и кнопки CTA, чтобы я мог легко прочитать и кликнуть.
6. Как посетитель, я хочу видеть небольшие декоративные искорки вокруг банки, чтобы промо выглядело энергично и соответствовало названию бренда «Cat Energy».
7. Как посетитель с отключённым JS, я хочу видеть статичное изображение банки без анимации, чтобы содержимое промо-секции оставалось доступным.
8. Как разработчик, поддерживающий проект, я хочу, чтобы vendor-файлы Three.js были удалены из репозитория, чтобы размер репозитория уменьшился.
9. Как пользователь с настройкой `prefers-reduced-motion`, я хочу, чтобы анимация отключалась или уменьшалась, чтобы страница не вызывала дискомфорта.

## Implementation Decisions

- **Удалить из `index.html`:** тег `<script type="module" src="js/cat3d.js">` и тег `<script src="js/vendor/gsap.min.js">` (gsap остаётся, но подключается другим способом — см. ниже).
- **Удалить из `source/js/`:** файл `cat3d.js` целиком.
- **Удалить из `source/js/vendor/`:** файлы `three.core.min.js`, `three.module.min.js` и любые другие Three.js vendor-файлы.
- **GSAP:** переместить `<script src="js/vendor/gsap.min.js">` в `<head>` (без `type="module"`, как обычный скрипт), чтобы GSAP был доступен глобально для `scripts.js` и новой анимации. Ничего нового не загружается — файл уже в `source/js/vendor/`.
- **`<canvas id="cat3d">`:** удалить элемент из HTML. `<picture>` с банкой продукта теперь всегда видим (без JS-скрытия).
- **Inline-SVG искорки:** добавить 5–6 `<svg>`-элементов непосредственно внутрь `.promo__image-box` (контейнер банки). Каждая искорка — 4-лучевая звезда (`✦`), реализованная как `<path>` в viewBox `0 0 20 20`. Размеры: от 10px до 18px. Позиционируются абсолютно относительно `.promo__image-box` в разных точках вокруг банки (сверху-справа, снизу-слева, по бокам). CSS-классы: `promo__sparkle promo__sparkle--N` (N = 1…6).
- **GSAP-анимация банки:** применяется к `.promo__image` (img внутри picture). Floating: `gsap.to` → `y: -14px`, `duration: 2.4s`, `ease: 'sine.inOut'`, `yoyo: true`, `repeat: -1`. Наклон: `gsap.to` → `rotation: 2`, `duration: 3.1s`, `ease: 'sine.inOut'`, `yoyo: true`, `repeat: -1`, `delay: 0.5`.
- **GSAP-анимация искорок:** каждая искорка — `gsap.to` → `scale: 1.35`, `opacity: 0.2→1→0.2`, `duration: 1.2–2.0s` (случайные задержки через `stagger` или индивидуальный `delay`), `yoyo: true`, `repeat: -1`. Эффект пульсации/мерцания.
- **`prefers-reduced-motion`:** перед запуском GSAP-анимаций проверить `window.matchMedia('(prefers-reduced-motion: reduce)').matches`; если true — анимации не запускать.
- **Вся новая логика** добавляется в `scripts.js` в виде отдельной секции после навигационного тоггла. Никакого нового модуля/файла не нужно.
- **Gulp-сборка:** не меняется (задача `copy` уже копирует `source/js/**`). Нужно только убедиться, что удалённые Three.js файлы убраны из `source/js/vendor/`.

### SVG-форма искорки (из прототипа — кодирует решение точнее, чем проза)

```svg
<!-- 4-point sparkle, viewBox="0 0 20 20" -->
<path d="M10 1 L11.5 8.5 L19 10 L11.5 11.5 L10 19 L8.5 11.5 L1 10 L8.5 8.5 Z"/>
```

Это единственная форма для всех 6 искорок; размер и позиция варьируются через атрибуты `width`/`height` и CSS `position: absolute`.

## Testing Decisions

Тестирование ручное, без автоматизированного фреймворка.

Хороший тест проверяет **внешнее поведение**, видимое пользователю: видна ли банка, двигается ли она, исчезли ли артефакты 3D.

Ручной чеклист для приёмки:

- [ ] Открыть `build/index.html`. В промо-секции видна банка (product image), не canvas.
- [ ] Банка плавно покачивается вверх-вниз — анимация запущена.
- [ ] Вокруг банки мерцают 5–6 искорок с разными задержками.
- [ ] Открыть DevTools → Elements: нет `<canvas id="cat3d">`, нет `<script src="js/cat3d.js">`.
- [ ] Открыть DevTools → Network: файлы `three.core.min.js` / `three.module.min.js` не загружаются.
- [ ] В DevTools → Console: нет ошибок.
- [ ] Открыть в браузере с `prefers-reduced-motion: reduce` (эмуляция в DevTools → Rendering → Emulate CSS media features) → анимации не запущены, банка статична.
- [ ] Отключить JS → банка видна статично, никаких сломанных элементов.
- [ ] Открыть `catalog.html` и `form.html` → промо-секции нет, ошибок в консоли нет.
- [ ] Проверить на мобильной ширине (375px) → банка и искорки позиционированы корректно.

Аналогов тестов в кодовой базе нет — проект без тестового фреймворка.

## Out of Scope

- Изменение текстового содержимого промо-секции («Функциональное питание для котов», слоган).
- Замена самого изображения банки на другое.
- Добавление новой иллюстрации или фотографии кота вместо 3D-модели.
- Анимация на других страницах (`catalog.html`, `form.html`).
- Переработка CSS-стилей промо-секции за пределами позиционирования искорок.
- Переход с Gulp на другой сборщик.

## Further Notes

GSAP уже подключён в проекте (`source/js/vendor/gsap.min.js`) и используется в `cat3d.js` через `window.gsap`. После удаления `cat3d.js` GSAP остаётся в сборке — его перемещение в `<head>` обеспечивает глобальную доступность `window.gsap` для `scripts.js`.

Inline-SVG искорки не требуют изменения `sprite.svg` или добавления новых файлов в `source/img/` — это сознательное решение, так как декоративные элементы нужны только на одной странице.

Удаление Three.js сократит `build/js/vendor/` примерно на 600+ КБ.
